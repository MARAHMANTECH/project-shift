# BRAND IDENTITY: Project SHIFT (The Soul of Mobility)

Denne fil fungerer som den ultimative visuelle guide for Project SHIFT. Alle UI-generationer (via Google Stitch) og frontend-kode skal overholde disse retningslinjer for at sikre en varm, menneskelig og moderne oplevelse.

## 1. CORE DESIGN PHILOSOPHY
- **Motto:** "Fra klinisk data til menneskeligt fællesskab."
- **Vibe:** Varm, organisk, troværdig og livsstilsorienteret (GoMore-inspireret).
- **NEVER:** Brug aldrig et sterilt "hospital-look", hårde kanter eller kold hvid/blå uden tekstur eller dybde.

## 2. VISUAL ELEMENTS
- **Colors:**
  - Base: Warm Sands (#F5F2ED) and Soft Off-white.
  - Accent 1: Forest Green (#2D5A27) - For natur og bæredygtighed.
  - Accent 2: Sun-kissed Orange (#FF8C42) - For energi og CTA'er.
- **Shapes:**
  - Border-radius: Minimum 16px på alle cards og knapper (Rounded-2xl/3xl i Tailwind).
  - Borders: Subtile borders (1px) med lav kontrast fremfor hårde sorte streger.
- **Dybde:** - Brug Glassmorphism (Backdrop-blur) på overlays og navigation.
  - Soft Layered Shadows: Brug `shadow-xl` med lavere opacity for at skabe naturlig dybde.

## 3. TYPOGRAPHY & CONTENT
- **Primary Font:** Satoshi eller Bricolage Grotesque (Moderne, venlig sans-serif).
- **Sprog:** Altid dansk i UI (æ, ø, å). Tonen skal være uformel, men professionel (f.eks. "Godmorgen, Lars" i stedet for "Bruger: Lars").
- **Icons:** Brug tykke, afrundede ikoner (f.eks. Lucide-react med stroke-width: 1.5).

## 4. UI COMPONENTS (Stitch Guidelines)
- **Cards:** Skal indeholde "white space" og bløde padding (p-6 eller p-8).
- **Forms:** Input-felter skal være store og lette at ramme på mobilen. Ingen standard browser-styles.
- **Visuals:** Integrér personlige elementer som profilbilleder og små illustrationer (sociale badges), der gør det menneskeligt.

## 5. ESG VISUALIZATION
- CO2-besparelser skal aldrig bare være tal. Brug visuelle metaforer: Træer der vokser, skyer der letter, eller en "Green Progress Bar" der føles som et spil (Gamification).
